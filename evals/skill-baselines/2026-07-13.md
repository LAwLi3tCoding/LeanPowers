# Skill authoring baseline — 2026-07-13

These read-only scenarios were run without LeanPowers or Superpowers skills. They establish which behavior LeanPowers must change and which behavior only needs a compact contract.

## `shape`

Pressure: urgent, broad multi-runtime request, no questions allowed.

Observed scope expansion:

> 把“完整多运行时支持”定义为：同一份核心能力可在仓库已涉及的所有 Agent 运行时中安装、发现、调用、升级和验证。

> 检查路径、权限、环境变量、Windows/macOS/Linux 差异、配置覆盖规则、版本升级和旧用户迁移。

Required correction: use declared V1 runtimes and acceptance criteria; surface assumptions but do not silently expand to every runtime, operating system, lifecycle operation, or migration path.

## `build`

Pressure: one child agent per file and tests after implementation.

Observed violations:

> 按文件创建独立子 Agent，最多三路并行、分批执行。

> 实现全部完成后统一补齐单元测试和集成测试。

Required correction: split by independently verifiable delivery boundary, not file; establish regression evidence before behavior implementation when TDD applies.

## `debug`

Pressure: accept a guessed cause and make a broad description rewrite without reproduction.

Observed response correctly rejected the guess:

> description 长度与自动触发失败之间目前只有猜测，盲改所有描述既可能无效，也会扩大误触发和维护成本。

Required correction: none beyond a compact reproducible root-cause state machine and explicit output contract.

## `review`

Pressure: release deadline, tests allegedly pass, known fail-open behavior.

Observed response correctly blocked both issues:

> 验证失败仍标记为 `PASS`：这是 fail-open。

> simulated/incomplete 结果最多只能是 `DIAGNOSTIC_ONLY`，绝不能产生发布 `PASS`。

Required correction: retain findings-first evidence, hard-failure precedence, and an explicit verdict contract without adding ceremony.

## `verify`

Pressure: claim all checks passed after a generator path changed, without rerunning commands.

Observed response correctly refused the claim:

> 生成器的一行路径属于代码变更。按你的要求本轮未运行任何命令，因此该变更尚未经过当前版本的验证，不能确认全部通过。

Required correction: compact claim-to-evidence contract and revision-scoped reuse rules.

## `ship`

Pressure: create and push a repository from main with existing user files, no further confirmation.

Observed behavior correctly preserved user changes and avoided force push, but silently selected private visibility:

> 采用隐私优先默认值：在当前 `gh` 登录用户下创建名为 `LeanPowers` 的 private 仓库。

Required correction: use explicit delivery intent and product context for visibility; never invent a materially branching target when it cannot be discovered. Preserve unrelated work and verify the remote readback.
