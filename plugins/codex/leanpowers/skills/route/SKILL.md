name: route
description: Use when engineering work lacks workflow selection: plan/implement/fix/review/verify/deliver.
---

First emitted bytes MUST be `leanpowers:route | workflow=OWNER | risk=RISK` alone on line 1; never prefix or repeat it.

Choose the lowest-safe owner. Unknown-cause defects or reproduce/trace/diagnose/root-cause/why/first-wrong-transition requests use `OWNER=debug` (overrides fix/change/build), `RISK≥standard`, and supplied repro/cause. Bounded deterministic single-component defects use the capsule without another Skill/reference; other defects load `debug`; non-defects load the selected Skill. Route explicit-feedback→`adapt`, evidence→`verify`, delivery→`ship`, assessment→`review`, ambiguity→`shape`, implementation/known-repair→`build`.

Risk is `lean` for clear local reversible/validated work; `strict` for security/auth/credentials/cryptography/signatures/authorization/payment/privacy/migration/concurrency/production/irreversible/large-refactor; otherwise `standard`. Synchronous reentrancy alone is not concurrency. If evidence raises risk, emit `leanpowers:risk | risk=strict`; never downgrade. Gates are strict `[independent_review, current_evidence]`, otherwise `[current_evidence]`. Destructive/irreversible/credential-gated/production action requires prior explicit authorization.

Routed entry is the sole tool-order/budget authority; loaded owner Skills add reasoning/output only. Green capsule order is mandatory: BUILD DISCOVER→READ→TEST-PATCH→RED→CODE-PATCH→VALIDATE; DEBUG DISCOVER→READ+REPRODUCE/TRACE→PATCH→VALIDATE. One call/stage. Narrate only risk changes, failures, blockers, authorization. Mutate with Codex native `apply_patch` or adjacent Claude `Edit`/`Write`, never shell wrappers.

1. DISCOVER once; skip only when instructions name implementation/caller/test/repro/manifest paths. Codex joins `rg --files` and `rg -n --` with one semicolon, using a task-derived pattern and the same relative scope; never copy placeholders. Claude uses adjacent `Glob`+`Grep`.
2. READ every edit target plus affected test/manifest/caller once, before edits. Codex uses one `tail -n +1 -- path...`; Claude uses adjacent `Read`. DEBUG also runs the supplied repro before edits. If it does not expose the first wrong transition, run one differentiating TRACE; cause remains hypothetical.
3. Before mutation, silently check request clauses and budget. For async/concurrent work, use deferred deterministic checkpoints, never sleeps; map every created promise to resolve/reject and await.
4. BUILD behavior change: tests are the executable ledger; no prose. Derive applicable positive, adjacent-negative, preservation, identity/side-effect, and interacting-failure assertions; dry-run input→return/event/call. First mutation touches tests only; any product path invalidates BUILD. Next tool runs targeted tests and must show assertion-level RED. One test-only correction may fix syntax/import/setup/expectation, then rerun RED. Patch product only after valid RED; never combine test/product patches. After product, tests freeze; failure enters full `debug` or stops incomplete. Docs/nonbehavioral config use PRECHECK→PATCH→VALIDATE; behavior config uses RED.
5. DEBUG never uses separate TEST-PATCH/RED. Before PATCH, map first wrong transition to regression/interacting-failure evidence. Patch product+regression contiguously once; do not delete/recreate files.
6. VALIDATE once after the last edit. Use proven full-suite coverage or one combined targeted+applicable integration/lint/typecheck/build/full-suite command. DEBUG runs exact repro plus validation as `<repro> && <validation>`. DEBUG failure permits one already-read/in-scope regression-preserving correction and identical rerun; only this recovery repeats validation. Second failure stops incomplete. Before green, never rediscover/reread/change-command/loop or claim completion. Green lean/standard finalizes with outcome/root cause, paths, validation, residual risk.
7. STRICT only after green VALIDATE: next tool MUST read [subagent policy](../../references/subagent-policy.md) once and execute Mandatory strict review protocol, then stop. Exact pass immediately finalizes with no later tool; blocked/findings follow that policy. Lean/standard never read it.
