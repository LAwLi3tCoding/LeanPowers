# Acknowledgments

LeanPowers is an independent project inspired by [Superpowers](https://github.com/obra/superpowers).

Superpowers demonstrated how coding-agent discipline can be expressed as reusable workflows rather than left to informal prompting. LeanPowers builds on that foundation, especially its commitment to:

- test-driven development and regression evidence;
- systematic debugging before repair claims;
- independent code review;
- verification before completion;
- safe use of worktrees, subagents, and branch delivery;
- explicit engineering process instead of unsupported confidence.

We thank Jesse Vincent and all Superpowers contributors for creating and openly sharing the project, documentation, and ideas that made LeanPowers possible.

LeanPowers explores a different optimization point: retain outcome-critical safeguards while making ceremony conditional on ambiguity and risk. It is not presented as an official Superpowers edition, a replacement verdict, or an attempt to diminish the upstream project. Comparisons in this repository are intended to document lineage, design tradeoffs, and evidence boundaries respectfully.

Superpowers and LeanPowers remain separate projects. References to Superpowers do not imply affiliation, sponsorship, or endorsement.

## 中文

LeanPowers 是一个受 [Superpowers](https://github.com/obra/superpowers) 启发的独立项目。

Superpowers 证明了：编码 Agent 的工程纪律可以被沉淀为可复用工作流，而不只是依赖临时提示词。LeanPowers 建立在这一基础之上，尤其继承并重视以下思想：

- 测试驱动开发和回归证据；
- 在宣称修复前进行系统化根因调试；
- 独立代码审查；
- 在宣称完成前执行验证；
- 安全地使用 worktree、子 Agent 和分支交付；
- 用明确工程流程代替没有证据的信心。

感谢 Jesse Vincent 和所有 Superpowers 贡献者创建并开放这个项目、文档与工程思想；没有这些基础，就不会有 LeanPowers。

LeanPowers 探索的是另一个优化点：保留影响工程结果的关键保障，同时让流程仪式根据歧义和风险按需触发。它不是 Superpowers 的官方版本，不以取代 Superpowers 为结论，也不试图贬低上游项目。仓库中的对比只用于尊重地说明项目谱系、设计取舍和证据边界。

Superpowers 与 LeanPowers 是相互独立的项目。仓库中的引用不表示双方存在隶属、赞助或背书关系。
