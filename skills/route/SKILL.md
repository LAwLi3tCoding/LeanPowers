First emitted bytes MUST be `leanpowers:route | workflow=OWNER | risk=RISK` alone on line 1; never prefix or repeat it.

OWNER capsule. Emit the declaration before task tools; this route invocation owns order through completion. Selected Skills add reasoning/output, never replace this order.

1. DISCOVER, then READ every later-edited existing target plus affected test, manifest, and caller. Preserve unrelated changes. Before PATCH, keep an in-context `Clause→test ledger:`: map each materially distinct `must|only|exact|preserve|reject` boundary to one positive or smallest distinguishing negative check; for representation or side-effect risk add `Counterexample: property=passing→nearby mutation→expected boundary`.
2. BUILD preserves DISCOVER→READ→TEST-PATCH→RED→CODE-PATCH→VALIDATE. Behavior uses tests first; product files stay locked until a focused test has meaningful RED: a new assertion fails for missing behavior, not syntax, setup, or unrelated failure. RED freezes it. Invalid test design restarts TEST-PATCH→RED on the pre-behavior baseline; never weaken assertions. Use getter/counter no-access, identity/snapshot immutability, guard-sentinel short-circuit, and one-property exact-boundary checks. Configuration/generated output uses a baseline/precheck; failing evidence is required only for behavior or defect.
3. DEBUG preserves DISCOVER→READ+REPRODUCE/TRACE→PATCH→VALIDATE. Reproduce/trace to the first wrong transition/root cause before editing, then prepare regression and product repair. Validate, then replay the exact reproduction; structured resolved output requires a separate final command. A launcher/tool-start error may correct only the command before supported validation runs and does not spend recovery. One failed supported validation permits either one immediate substantive correction to already-read in-scope paths followed by the identical command, or block/rescope. No intervening task tool, test weakening, empty retry, second correction, or second failure.
4. After the final edit, run targeted regression plus applicable integration, lint, typecheck, build, package, or full-suite validation. First green freezes the completed acceptance set; a material omission starts a new incomplete cycle. File edits invalidate evidence; read-only reporting does not.
5. STRICT after final green starts a fresh independent read-only reviewer with the exact task, ledger, changed paths, and current validation; wait for completed Review YAML and require PASS. Findings reopen repair→green→fresh review. No mutation follows PASS.

Choose the lowest-safe owner. Unknown-cause defects or reproduce/trace/diagnose/root-cause/why/first-wrong-transition requests use `OWNER=debug` (overrides fix/change/build), `RISK≥standard`, and supplied repro/cause. Bounded deterministic single-component defects use the capsule without another Skill/reference; other defects load `debug`; non-defects load the selected Skill. Route explicit-feedback→`adapt`, evidence→`verify`, delivery→`ship`, assessment→`review`, ambiguity→`shape`, implementation/known-repair→`build`.

Risk mirrors the classifier. `lean` only when `clear`, `local`, `reversible`, and `establishedValidation` are true and no standard or strict signal applies. `causeKnown=false` is `standard`; so are `preferredMode=standard`, `behaviorChange`, `boundedUncertainty`, `dataModelChange`, `defect`, `dependencyChange`, `diagnosisRequested`, `externalSystem`, `multiFile`, `publicBoundaryChange`, `scopeExpanded`, and `validationFailed`. Use `strict` for `preferredMode=strict`, `authorization`, `authentication`, `concurrency`, `credentialGated`, `credentials`, `cryptography`, `dataRisk`, `destructive`, `irreversible`, `largeRefactor`, `migration`, `payment`, `privacy`, `production`, `reviewHighRisk`, `security`, `secrets`, and `signatureVerification`. Synchronous reentrancy alone is not concurrency. If evidence raises risk, emit `leanpowers:risk | risk=strict`; never downgrade. Gates are strict `[independent_review, current_evidence]`, otherwise `[current_evidence]`. Destructive/irreversible/credential-gated/production action requires prior explicit authorization.

Routed strict loads only the [subagent policy](../../references/subagent-policy.md) after green and finishes under it. Lean/standard never read it. Mutate with Codex native `apply_patch` or adjacent Claude `Edit`/`Write`, never shell wrappers.
